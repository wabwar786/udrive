# Phase 8 security and current limitations

## Implemented safeguards

- OTP values are stored as HMAC hashes, not plaintext.
- Refresh tokens are random and stored as hashes.
- Access tokens expire quickly and refresh tokens rotate.
- Token versioning invalidates stale access tokens after approval, suspension or rejection.
- CNIC and licence identifiers are stored as keyed hashes with masked display values.
- Verification evidence is not publicly exposed as static files.
- Uploads validate extension, size and file signature.
- File paths are generated by the server and protected against traversal.
- Admin endpoints require Admin or Operations role.
- Approval prerequisites are enforced server-side.
- Android cleartext HTTP traffic is disabled.

## Before public launch

- Replace the development OTP provider with a real SMS provider.
- Remove the fixed `1234` OTP from testing environments.
- Move verification evidence to private S3-compatible object storage with signed Admin access.
- Add malware scanning for uploaded files.
- Add Admin multi-factor authentication and preferably secure HttpOnly cookie sessions for the web portal.
- Add privacy policy, consent, retention and deletion workflows for identity documents.
- Add rate limiting per phone, IP, device and account using Redis.
- Add automated session cleanup, OTP cleanup and refresh-token reuse alerts.
- Complete penetration testing and legal review before collecting real CNIC/licence documents.

Phase 8 is suitable for controlled staging and functional testing. It is not yet the final public-production identity system because live SMS, private object storage and formal compliance processes remain pending.
